Blog update

2020 June 23rd

When I created this site, I was thinking about making a simple CMS by myself (just for fun). Since all my posts are just simple content, a database is not necessary, which means a Flat-File CMS is a good choice. It's simple, effective, and easy to set up. For the first few pages, because of my laziness, I just hard-coded the link of my posts. But now I got some free time on weekend, I will try to add more functions to this site. Back to topic, my approach to add CMS functions:
  • Blog folder structure: all of my posts are divided into folders which are named by the year they are created.
  • My first stupid thought: loop all the folders, in each folder, get all the file names, not really efficient I must say.
  • So I need a place to store a list of my posts, I decided to make it simple: use a json file. Whenever I create a new post, I just add an item to the json file.
  • My web will load the file only once at startup. (Maybe I should think how to reload the file every time I modify it without restart the web?)
  • Final is adding the paging.
